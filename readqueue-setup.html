ReadQueue — Quick Add Setup
Set up one-click saving from your browser and iOS share sheet.
Phase 1 of 3
Deploy your backend endpoint
1

Create a free Cloudflare account

Go to dash.cloudflare.com/sign-up and create a free account. No credit card needed. The free tier supports 100,000 requests/day — more than enough.

2

Create a new Worker

In the Cloudflare dashboard: click Workers & Pages in the left sidebar → Create → Create Worker → name it readqueue → click Deploy.

You'll land on an editor page. Click Edit code, then replace ALL the default code with the code below.

/* paste generated worker code here — fill in your details below first */
3

Fill in your details to generate the Worker code

Enter your keys below and click Generate — this produces the exact code to paste into Cloudflare.

4

Deploy and note your Worker URL

After pasting the code, click Deploy (top right). Your Worker URL will appear at the top — it looks like https://readqueue.YOUR-SUBDOMAIN.workers.dev. Copy it and paste it below.

5

Make sure your Google Sheet allows API writes

Your Sheet sharing must be set to "Anyone with the link — Editor" for the Worker to write to it (the Worker uses the API key, not your login). This is separate from the ReadQueue app itself, which uses your Google login.

Open your ReadQueue Google Sheet → Share → change to "Anyone with the link" → Editor.

Your ReadQueue app login is still protected — only authorised Google accounts can open the app. The Sheet API key just allows the Worker (running on your server) to append rows.
Phase 2 of 3
Install the browser bookmarklet
1

Show your bookmarks bar

In Chrome: Ctrl/Cmd + Shift + B. In Safari: View → Show Favorites Bar.

2

Drag this button to your bookmarks bar

Click and drag the button below up into your browser's bookmarks bar. That's it — it's installed.

📚 Save to ReadQueue
Drag this button to your bookmarks bar

On mobile Safari: tap the share icon → Add Bookmark → name it "Save to ReadQueue". Then find it in your bookmarks when you want to save a page.

3

Test it

Navigate to any article, click the bookmarklet. A small toast notification will appear confirming the item was added to your ReadQueue.

Phase 3 of 3
Install the iOS Share Sheet shortcut
1

Download the Shortcut

On your iPhone or iPad, tap the button below. It will open the Shortcuts app and ask you to add the shortcut. Tap Add Shortcut.

ReadQueue — Save to Queue

Adds the current page to your ReadQueue via the iOS share sheet. Works in Safari, Chrome, and any app that shares URLs.

📱 Add to Shortcuts

If the Shortcut link isn't working yet (it needs your Worker URL above), you can also create it manually — see step 3 below.

2

Add it to your Share Sheet

After adding the shortcut: open Safari, tap the Share button → scroll down → tap Edit Actions → find "ReadQueue" → tap the + to add it. It will now appear every time you share a URL.

3

Or create it manually (if the link doesn't work)

Open the Shortcuts app → tap + → Add Action → search for "Get URLs from Input" → add it. Then add a second action: search "Get Contents of URL" → set Method to POST → URL to your Worker URL → add headers:

URL: [your worker URL] Method: POST Headers: Content-Type: application/json X-Worker-Secret: [your worker secret] Body (JSON): { "url": [Shortcut Input], "title": "" }

Set the shortcut to receive Safari web pages as input so it appears in the share sheet.

Test your setup
Send a test URL to your queue

Paste any URL below and click Send — if everything is set up correctly, it will appear in your ReadQueue within a few seconds.
